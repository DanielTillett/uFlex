Understanding the Built-in Console

Inside the uFlex there is an object called console. The console tracks and logs proccess, steps and errors inside the class object. There are three main sections inside the console: report, error, form.

report - It holds every proccess and steps inside the class including the error and form.

View Code 
    Array
    (
        [0] => Passwords matched
        [1] => No Error Found!
        [2] => New Validation Object added
        [3] => The username is Valid
        [4] => The password is Valid
        [5] => The email is Valid
        [6] => No Error Found!
        [7] => Error: This Email is Already in Use
        [8] => Error: This Username is not available
        [9] => 2 Error(s) Found!
    )

error - It store the errors that makes a proccess fail or return false. If an inside function returns false the detailed error will be here so be sure to check it out.

View Code 
    Array
        (
            [0] => This Email is Already in Use
            [1] => This Username is not available
        )

form - It holds the name of invalid fields, fields that did match with the regular expression or are not within the range limit in length. This comes very handy to notify users of invalid specially if using AJAX with a javscript framework like jQuery.

View Code 
    Array
    (
        [0] => email
        [1] => username
    )

The prints from above are not directly from the console object itselft as the console is a little more complex. Pretending the $user is the uFlex object this is how the clean prints from above are gotten:

View Code 
    	//Reports
        $user->report();
        
        //Erros
        $user->error();
        
        //Form
        $user->form_error();

Above you can clearly see the three mayor sections report, error and form, you can also notice that all have common sub-sections. This sub-sections simply indicate the proccess in which the the step or error was caughted. There are currently three possible sub-sections or proccess: login, registration and update.

Going back to the clean prints all the way to begining, they look clearer because using such method only gets the last proccess logged, in this case the registration. Here is how you can get a clean print for report, error and form in a single object:

View Code 
        $data['report'] = $user->report();	
        $data['error'] = $user->error();
        $data['form'] = $user->form_error();

If you print the $data array you will get this:
